This example makes use of sample source code that is part of your Enterprise Architect installation.<br/>The sample code for this example is installed to:  "C:\Program Files (x86)\Sparx Systems\EA\Code Samples\VEA"<br/>but should be copied to a different location, so that you don't need to run with elevated permissions in order to compile and build the project.<br/>So, for example, copy everything in "C:\Program Files (x86)\Sparx Systems\EA\Code Samples\VEA", <br/>to "C:\EA_CodeSamples\VEA".<br/><br/>Before running the example, you must configure Local Path definitions for %VEA% and %MSVS%.<br/>-    %VEA% = "C:\EA_CodeSamples\VEA"    (the directory into which you copied the sample code)<br/>-    %MSVS% = "C:\program files (x86)\Microsoft Visual Studio 9.0\Common7\IDE"<br/>(From the 'Develop' ribbon, select the option 'Preferences #gt; Options #gt; Configure Local Paths'.)<br/><br/>1.  Open the Execution Analyzer window and Workbench window.<br/>2.  Locate the Analyzer script 'MyClassLibrary'.  It can be found in the Execution Analyzer window under 'Visual Execution Analysis..Net.Workbench'.<br/>3.  Select the script (by placing a checkmark), then build the project by clicking the 'Build' button.<br/>4.  Create an instance of a class by selecting CRobert on <font color="#00008b"><b>this </b></font>diagram and pressing Ctrl+Shift+J.<br/>5.  When prompted, enter a Name, then click on 'Create'.<br/>6.  When prompted, click on 'Invoke'.<br/>7.  The Workbench window will now have an instance of this Class.<br/><br/>
