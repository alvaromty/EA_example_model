With the aid of a UML profile for data modeling, a physical database schema can be modeled as a Class diagram.<br/>A variety of Database objects can be modeled including tables, foreign keys, views and stored procedures.<br/>Data Definition Language can then be generated directly from the model for a large number of supported database systems.<br/>
