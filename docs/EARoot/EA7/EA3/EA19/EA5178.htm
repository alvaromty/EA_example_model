Appears In: Properties of Requirements
The effective management of requirements is critical to the success of any project.<br/><br/>The properties of Requirement elements allow us to make use of a range of tools in Enterprise Architect that help you to manage those requirements.<br/><br/>All Enterprise Architect elements have a number of standard properties including Status, Author and Phase.  The Requirement element has two additional properties, <b>Priority</b> and <b>Difficulty</b>.  Some requirement processes might specify particular properties that are not part of the standard list, such as Owner and Volatility (Stability); these can be applied using Tagged Values that can be set for each Requirement element.<br/><br/>In managing requirements, you could make use of the Model Search facility to retrieve high priority requirements and define work packages for the implementation teams, or you might retrieve requirements for a particular phase of the project.  Search results can be used as input for report generation, so you could generate a report on all requirements that are 'owned' by a particular stakeholder.  Filters can be employed with diagrams and element lists, allowing you to hide or to highlight requirements that meet specific criteria.<br/><br/>To view or update the properties of any element, select the element either in the Browser window or on a diagram, then from the 'Design' ribbon, choose the option: 'Element #gt; Editors #gt; Properties' (or 'Tagged Values)'.<br/>
