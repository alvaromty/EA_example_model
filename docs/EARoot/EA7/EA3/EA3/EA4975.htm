The Boundary element is particularly useful for defining what is inside a system and what is outside. It is derived from the Use Case system boundary, but can be used extensively in other contexts as a way of describing the separation between a system (or part of a system) and its external environment.<br/>Any number of Boundaries can be added to a diagram, with other elements such as Use Cases, Features or Requirements placed inside the boundary.  Boundary elements can also be nested within one another, using different fill colors to show the different levels.<br/>The Boundary can be configured to have a number of vertical and horizontal swimlanes, which are useful for grouping elements into different sections of the boundary.<br/>They can also be set to have a variety of different styles which allow you to customize their shape where required.<br/>Boundary elements can be found on the 'Common' toolbox page.<br/>
