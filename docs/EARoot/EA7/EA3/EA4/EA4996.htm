The Data Flow Diagram (DFD) is part of the Structured Systems Analysis and Design Method.  It is one of the three essential perspectives of Structured Systems Analysis and Design Method SSADM.<br/><br/>DFDs are used to illustrate the way that data flows through a system, showing the Processes, Data Stores and Entities that relate to the data.<br/><br/>With a data flow diagram, users are able to visualize how the system will operate, what the system will accomplish and how the system will be implemented. Developing a DFD helps in identifying the transaction data in the data model.  A data flow diagram can also be used for the visualization of data processing (structured design).<br/><br/>It is common practice for a designer to draw a context-level DFD first, which shows the interaction between the system and outside entities.  This context-level DFD is then "exploded" to show more detail of the system being modeled.'<br/><br/><br/>
