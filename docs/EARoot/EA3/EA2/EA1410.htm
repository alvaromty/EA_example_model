Activity diagrams are used to model system behaviors, illustrating the interactions between the various elements of a system. They show the logical paths that a system follows, based on conditional tests, concurrent processing, data access, and process interruptions.  <br/>An Activity diagram's elements and connectors are created from the 'Activity' pages of the Diagram Toolbox.<br/>These buttons link to examples of Activity diagrams that use a number of the features added to Activity diagrams in UML 2.0. <br/>
