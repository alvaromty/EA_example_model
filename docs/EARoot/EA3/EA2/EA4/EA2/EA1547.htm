Scenario Type
Basic Path Basic Path
Details:
1. Use case begins when the Client selects the "Create Account" command.<br/>2. The Client enters name, address, e-mail address, password and password confirmation.<br/>3. Client presses Submit button.<br/>4. Submitted details are validated.<br/>5. Processing message is displayed to the Client.<br/>6. New account is created in the database.<br/>7. Success message is displayed to Client.<br/><br/>
No Submit Alternate
Details:
The Client may press the Exit or Back button at step 3. The use case terminates with no action performed.<br/>
Validation fails Alternate
Details:
At step 4, if validation fails then an error message is displayed and the Client is returned to step 2 with the invalid field or fields highlighted.<br/>
Cannot Create New Account Alternate
Details:
In step 6, if the database create fails, the Client is informed and the use case terminates.<br/>
