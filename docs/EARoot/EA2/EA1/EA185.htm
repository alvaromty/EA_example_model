Custom tables can be used to display non-modeled data of various kinds.  This diagram contains a number of examples of custom tables.<br/><br/>To manually edit the table data, you must place the table into 'Edit' mode.  To do so, select the table, then press 'F2' or right-click and choose 'Edit Custom Table'.  To exit edit mode, press 'Esc' or right-click and choose 'Exit Edit'.<br/><br/>Custom Tables may also define an operation named 'script'.  The script can be executed by right-clicking and choosing 'Execute behaviour script'.  The script can update cell data within the table through the use of API functions.<br/><br/>To view and edit the 'script' code, choose either 'Design #gt; Element #gt; Behavior' or 'Develop #gt; Source Code #gt; Behavior' from the ribbon.  A code editor window will open, where you can enter the required scripting code.<br/><br/>The second example on this diagram, 'Example with script', defines such a script.  When the script is run, the data in the cell at position (1, 1) is updated to show the date-time at which it's data was updated.<br/>
