This diagram demonstrates how to define the valid relationships between the types defined in a profile.<br/><br/>The two connector types created in this profile demonstrate the different ways that the relationships can be specified.<br/><br/>In the first example, individual pairings of valid relationships are defined using the stereotyped relationship connector between the types that need to be linked. As of version 15, the stereotype property for this relationship is available in the docked properties window.<br/><br/>A stereotyped relationship is recommended when:<br/><ul>
<li>Only a single source and target is possible for a stereotyped connector</li><li>Different source elements require different sets of target elements.</li></ul><p/>
