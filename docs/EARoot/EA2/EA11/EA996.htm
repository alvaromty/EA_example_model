Appears In: Model Based Add-Ins
This diagram shows two examples of Model Based Add-Ins.  The Add-Ins can be enabled by selecting the ribbon option 'Specialize #gt; Add-Ins #gt; Manage-Addin', then placing a tick in the checkbox 'Load on Startup' beside the Add-In that you want to enable.<br/>Model Based Add-Ins are defined within the model, using stereotyped classes.  In this case, the stereotype is 'JavascriptAddin'.  Using these stereotyped classes, you can specify Receptions, Methods and Properties that together define the behaviors of the Add-In, and how it responds to the various events occurring within the system.<br/>Receptions are defined by specifying a signal that will be received.  For this reason, the signals that are relevant to Model Based Add-Ins should be included within the model where you are defining or using Model Based Add-Ins.  You will find the signals that are relevant to Model Based Add-Ins listed on the diagram 'Broadcast Types'.<br/>The Receptions also provide a place to specify JavaScript code that will be executed in response to the receipt of a corresponding signal.  The reception code can also call functions defined as Methods of the class, while the class attributes can be used to define global variables that are available to the executing code.<br/>
