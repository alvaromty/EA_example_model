Appears In: Reusable Assets Service
Reusable Assets are held in a remote registry, accessed through a Cloud connection. The registry contains any number of Storages, which can either be left unprotected for any user to create, or change-restricted by an Administrator password. Each Storage can contain any number of Asset Packages holding modeling structures, and files containing textual or graphical information and data. When a user creates a Storage, they can protect the contents from being updated in the Registry or downloaded into a model, using password protection defined by that user.<br/>Reusable Assets consist of:<br/><ul>
<li>Packages containing elements, diagrams and structures (as drawn from any point within the Browser window), and</li><li>Files in a range of text, code and graphic formats, including .eap files</li></ul>
An asset can be, for example:<br/><ul>
<li>A common Class library or framework</li><li>A set of common Requirements or Use Cases</li><li>A draft specification document</li><li>Marketing collateral</li></ul>
The RAS gives distributed teams convenient access to a single 'source of truth'  for shared data, including project milestones, architectural frameworks and industry standards.<br/>
