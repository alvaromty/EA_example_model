Document Artifact
A Document Artifact is an element that is used to store an RTF document, directly inside the model repository.  These documents are created using Enterprise Architect's built in document editor, which provides most of the functionality you would find in any fully fledged word processor.

Like any element in your model, Document Artifacts can be linked to other elements, so they provide a means by which additional documentation can be associated with any other element.  However, they also offer the benefit of being able to create hyperlinks within the document, that link to other model elements or diagrams.

Here are some examples of hyperlinks to other model elements, within the document:
  • Artifacts Diagram 
  • Requirement - Manage Inventory
  • Relationship Matrix

An analyst will often be required to create structured documentation over and above what might properly be placed in an element's notes or in a corporate document repository.  This could include things like interview plans, focus groups agendas and surveys.  There is rarely a requirement to store this type of documentation in a corporate document repository, so storing them inside the modeling repository is an ideal solution.


Artifact (base)
The (base) Artifact element provides a means for connecting elements in the repository with external files or web resources.
When a file or resource is dragged onto a diagram, a menu will prompt the user to select the type of Artifact to create:
  • 'Hyperlink' creates a Hyperlink element on the diagram; you can select a sub-option to define what happens when double-clicking on the Hyperlink: 'Open' displays the file content and 'Edit' opens it within the assigned file editor
  • 'External' creates an Artifact element on the diagram that stores a hyperlink to the external file; press F12 or Ctrl+E to open the external file
  • 'Internal' creates an Artifact element on the diagram and makes a copy of the file inside the repository; double-clicking the Artifact launches the internal document
  • 'Insert' (graphics files only) inserts the file into the diagram as a filled Boundary element


You can also create Artifact elements by dragging one of the 'Artifact' icons from the 'Artifacts' page of the Diagram Toolbox.

The 'Common' page of the Toolbox also has a generic 'Artifact' icon.   When you drag it onto a diagram, you are offered a choice as to which type of Artifact to create.


Other Artifacts
The 'Artifacts' tool box page offers a range of other artifact types that can be created and placed on a diagram.  Double-clicking the artifact on the diagram will typically open or run the associated specification.


They include:

  • Matrix Specification - encapsulates a saved Relationship Matrix profile.
  • Report Specification - encapsulates a saved Document Generation specification
  • Model View - represents a table of model data, retrieved using a custom SQL search
  • Standard Chart - an artifact that records information for generating a bar chart or pie chart from data in your model.  Once you have defined the chart, it automatically displays with the latest information whenever you open the parent diagram.
  • Time Series Chart - an artifact that records information for generating a linear graph of model properties over time.  Once you have defined the chart, it automatically displays with the latest information whenever you open the parent diagram.
  • Executable State Machine - an artifact that specifies the objects that will be involved in a simulation.
  • SysMLSim Configuration - records a specific SysML simulation configuration
  • Image Asset - an artifact used to store an image in the model repository
  • Reading List - provides an ordered list of elements.  Information from each element is displayed in the Document Window
  • Checklist - facilitates creation of a list of items, each with a checkbox
  • User Story - provides a linked document, for recording a Business Use Case.
  • Working Set - a saved profile that specifies a workspace layout.
